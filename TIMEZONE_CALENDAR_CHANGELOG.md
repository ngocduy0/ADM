# Timezone & Calendar Fix Changelog

## Booking time correction

- Fixed the `19:00 -> 12:00` error caused by writing a UTC ISO instant into a Postgres `timestamp without time zone` booking column.
- New bookings now store the literal Vietnam business wall-clock value.
- Added a backward-compatible marker in the internal booking notes payload.
- Existing legacy bookings without the marker are interpreted as stripped UTC and converted back to `Asia/Ho_Chi_Minh`.
- Internal markers are removed before notes are shown to administrators or customers.

## Notification correction

- Notification text is rebuilt from the current reservation instead of trusting stale stored text.
- Notification order uses the timestamp embedded in generated reservation IDs when available.
- Supabase timestamps without a timezone suffix are treated as UTC, not browser-local time.
- Duplicate trigger/application notifications are collapsed to one notification per reservation.
- Mark-as-read updates all database notification rows that share the same reservation ID.
- Editing a booking refreshes its notification details immediately.

## Calendar redesign

- Added a seven-day selector with booking counts.
- Added a complete selected-day agenda sorted by arrival time.
- Each row clearly shows time, customer, phone, venue, table/area, guest count and status.
- Added unique status colors for New, Contacted, Confirmed, Completed, Cancelled and No-show.
- Added a full week overview with every booking grouped and sorted inside its day.
- Booking detail and edit actions remain available from calendar cards.

## Console notes

`onboarding.js`, `content-script.js` and `WidgetId` messages shown in Coc Coc DevTools are injected by browser extensions and are not generated by this application. The stale `/_next/static/css/... 404` warning is normally caused by an old development `.next` cache; this ZIP excludes `.next`.
